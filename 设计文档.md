# 《Pitch Invader: World Cup Craze》（狂飙世界杯：冲场之王）— 游戏设计文档

> 一款 HTML5 Canvas + 原生 JavaScript（无依赖、可单文件运行）的 2D 俯视角像素街机游戏。
> 在线试玩：https://jackliu-jin.github.io/pitch-invader/ ｜ 调试版：https://jackliu-jin.github.io/pitch-invader/debug.html

---

## 1. 游戏概述

| 项目 | 内容 |
|---|---|
| 类型 | 2D 俯视角 / 躲猫猫追逐 / 街机刷分 / 含 Roguelite 要素 |
| 主题 | 玩家扮演一名疯狂球迷，冲入世界杯决赛（阿根廷 vs 葡萄牙）的球场 |
| 核心情绪 | 高压下的刺激感、出名后的狂热、人生巅峰的躁动 |
| 平台 | 网页（PC 键盘 + 移动端虚拟摇杆，自适应横屏） |
| 视觉风格 | 像素 Q 版写实，俯视球场含完整看台与球场标线 |

### 设计思路
游戏把"冲场球迷"这一现实中转瞬即逝的荒诞事件，放大成一段"高光—躁动—被捕"的情绪曲线。核心动词不是"战斗"而是"合影 + 逃跑"——玩家既是猎手（追球员合影），又是猎物（被保安追捕）。这种**双向追逐**是整个玩法的支点：你越疯狂刷分，保安越多，风险越高，但出名带来的快感也越强。

---

## 2. 目标与胜负条件

- **核心目标（无尽模式）**：尽可能多地与场上球员合影刷分，活得越久、合影越多越好。
- **失败条件**：被任意保安碰到即被逮捕，游戏结束。
- **没有"通关"**：早期版本是"集齐 22 人即胜利"，后改为**无尽刷分**——球员源源不断登场，挑战上限只取决于玩家能撑多久。结算展示：得分、合影人数、存活时间。

### 设计思路
把"有限通关"改成"无尽刷分"是为了解决**后期空旷无聊**的问题（详见 §6.2）。失败只有一种（被捕），干净利落，强化"在刀尖上跳舞"的紧张感，也契合"冲场迟早被抓"的现实荒诞。

## 3. 操作控制

| 操作 | PC 键盘 | 移动端 |
|---|---|---|
| 移动（360° 无极） | WASD / 方向键 | 左侧虚拟摇杆 |
| 冲刺（Sprint） | 按住 Shift | 按住右侧"冲刺"按钮 |
| 翻滚（Roll） | 空格 Space | 点击右侧"翻滚"按钮 |

- 摇杆为"动态半径"实现，触点拖动方向即移动方向，松手归零。
- 进入游戏时尝试请求全屏 + 锁定横屏；竖屏时显示"请横屏"提示遮罩。

### 设计思路
左手移动、右手两个动作键是移动端追逐游戏的标准布局，拇指零冲突。冲刺"按住"、翻滚"点击"区分了"持续资源消耗"与"瞬时爆发"两种手感。

## 4. 角色系统

### 4.1 主角 · 冲场球迷（The Invader）
- 黄衣像素小人，从球场底部边线翻入场内（写实出生点）。
- 属性：移动带惯性、可冲刺、可翻滚（带无敌帧）、有体力条。
- 这是唯一受玩家直接控制的单位。

### 4.2 球员（Football Players）
分两阵营，场上恒定维持 **14 人**（无尽补充）：

| 类型 | 占比 | 特征 | 合影得分 |
|---|---|---|---|
| 普通球员 | 多数 | 逃跑触发半径小（100px），体力小（55），容易追到累垮 | 150 |
| 大牌球星（10 号 / 7 号） | 同时最多 2 名 | 逃跑触发半径大（175px），体力大（170），极难抓 | 400 |

- 阿根廷：蓝白配色，球星 10 号。葡萄牙：红绿配色，球星 7 号。
- **所有球员都会躲避**：玩家靠近就逃，头顶显示体力槽（球星的槽更宽更醒目）。
- 合影成功后球员跑下场离开，由系统从边线补入新球员，保证球场始终热闹。

### 4.3 保安（Security）
- 黑衣，从球场四周边界生成，径直朝玩家寻路。
- **速度永远慢于主角**（按主角当前速度的比例计算，普通 0.62×），但**数量会不断膨胀**形成"尸海"。
- 惯性大（转向笨重），玩家可用急停 / 变向假动作把他们甩开。
- 靠近玩家会发动**蓄力飞扑**（见 §5.6）。

### 4.4 精英保安 / 安保队长（Elite Security）
- 紫衣 + 金色 ★ 标记，后期出现。
- 更快（0.85×）、惯性略小（更难骗）、会**预判玩家走位**（朝玩家速度前方拦截）。

### 4.5 暴动球迷 NPC（Riot Decoys）
- 绿衣，触发"暴动时间"时从场外冲入 3–5 名，在场上乱跑。
- 本身不可合影、不威胁玩家，作用是**吸引保安**（见 §5.7）。

### 设计思路
五类角色构成清晰的**威胁/奖励梯度**：普通球员（低风险低回报）、球星（高风险高回报）、保安（持续压力）、精英（后期尖峰压力）、暴动 NPC（喘息红利）。颜色编码（黄=你、蓝白/红绿=目标、黑/紫=威胁、绿=友军）让玩家在混乱画面中瞬间读懂局势。

## 5. 核心机制详解

### 5.1 合影机制（Selfie）
- 靠近球员（合影半径 50px，可升级扩大）触发合影进度条（约 60 帧 ≈ 1 秒填满）。
- **贴脸抓捕**：距离 < 40px（合影半径 × 0.8）时直接"抓住"对方，强制定身合影，进度更快（1.3×）。
- 普通球员只要靠近且对方未在逃跑（或已累垮停下）即可拍；球星几乎只能靠贴脸抓住。
- 合影成功：屏幕爆闪白光（球星金光）、屏幕微震、爆发纸屑、人声欢呼、随机解说旁白。
- **追逐加成**：刚刚追过（逃跑过）的目标合影得分 ×2。

### 5.2 体力机制（Stamina）— 主角
- 绿色体力条显示在头顶与底部 HUD。
- 冲刺持续消耗（0.4/帧），翻滚瞬时消耗（默认 5）。
- 停止消耗后缓慢恢复（0.45/帧）。
- **力竭惩罚**：体力归零后进入约 2 秒（120 帧）"气喘吁吁"状态，移速降至 40%，结束后回复少量体力。

### 5.3 冲刺（Sprint）
- 按住时移速 ×1.8，持续吃体力。用于追球星、甩保安。

### 5.4 翻滚（Roll）
- 瞬时消耗体力，朝当前移动方向突进一段（速度 9，持续 14 帧）。
- **无敌帧（I-frames）**：突进期间可穿过保安不被抓；翻滚中也无法被球员"抓住合影"（双向无敌）。
- 翻滚后有短冷却（20 帧）。是化解飞扑、贴脸逃生的关键技。

### 5.5 惯性 / 假动作（Inertia）
- 主角、球员、保安都采用"当前速度向目标速度平滑过渡"的惯性模型，而非瞬间贴地。
- 惯性强度分层：**保安最重 > 球员 > 主角最灵敏**。
- 玩家可用急停、突然变向制造"假动作"，让惯性大的保安冲过头，拉开距离。

### 5.6 保安飞扑（Lunge）三段式
靠近玩家（70px 内）时，保安进入状态机：
1. **蓄力 charge（28 帧）**：原地下蹲，脚下浮现红色警示圈——提示"要扑了，快躲"。
2. **飞扑 lunge（16 帧）**：朝玩家预判位置高速突进（2.4×），身体伸展。
3. **倒地恢复 recover（45 帧）**：扑空后爬起，移速骤降——这是玩家**反打/反向甩开**的窗口。
- 之后进入冷却（80 帧）才能再次飞扑。

### 5.7 煽动球迷 / 暴动时间（Crowd Riot）
- 玩家有"狂热值/连击槽"，合影累积（球星 +35、普通 +18），短时间连续合影更快攒满。
- 满值触发**【暴动时间】**：场外冲入 3–5 名绿色 NPC，持续约 8 秒（480 帧）。
- 期间保安会被附近（260px 内）的暴动 NPC 吸引，转而去追他们，为玩家争取大量喘息与合影时间。

### 5.8 Roguelite 升级
- 得分达到阈值（初始 1000，之后每次 ×1.7 递增，无尽延续）时，弹出升级面板（游戏暂停，类"子弹时间"）。
- 从 4 选项中随机给 2 个二选一：

| 选项 | 效果 |
|---|---|
| 体力上限 | +30 |
| 移速 / 冲刺速度 | +15% |
| 翻滚消耗 | −1 |
| 合影判定半径 | +20% |

- 选择后右上角等级 HUD 实时更新 + 金色解说提示 + 金光反馈，让升级"看得见"。

### 5.9 无尽球员池（Endless Pool）
- 场上恒定 14 名球员；合影后的球员跑下场移除，系统立即从边线补入新球员。
- 球星稀有：同时最多 2 名，被拍走后有冷却（360 帧）才再刷新。
- 解决"后期球场空旷无聊"，保证全程画面热闹、目标不断。

### 5.10 动态难度（Escalation）
保安目标数量随多重因素增长，形成持续升级的压力曲线：
```
保安目标数 = 2 + 存活时间/15 + 得分/1000 + 移速升级加成 + 合影人数/6   （上限 32）
```
- 生成间隔也随时间、合影数缩短；后期还会掺入会预判的精英保安。
- 设计意图：你刷得越猛、升级越快、活得越久，保安就越多越凶——天花板由玩家自己的贪婪决定。

## 6. 关键设计决策与思路

### 6.1 为什么"所有球员都逃 + 体力槽"
最初只有球星逃跑，普通球员站桩，追逐感单薄。改为**全员躲避 + 体力机制**后：
- 普通球员体力小，追几下就累垮、露出可合影的"破绽窗口"——节奏快、爽快。
- 球星体力大（约 3 倍），必须靠冲刺贴身、翻滚抢位才能抓住——形成高难高回报目标。
- 体力槽把"还能不能跑"可视化，玩家能读出"再逼一下他就垮了"的博弈信息。

### 6.2 为什么改成无尽模式
"集齐 22 人通关"会导致越打越空、收尾乏味。无尽池 + 动态难度让游戏变成**自定义难度的刷分跑分**：想稳就保守，想冲就贪。可玩性大幅延长。

### 6.3 为什么保安要"慢但多 + 大惯性 + 飞扑预警"
- **慢**：保证玩家有操作空间，不会被瞬间追死，挫败感低。
- **多（尸海）**：用密度而非速度制造压迫，后期走位空间被逐渐压缩。
- **大惯性**：开辟"假动作"操作深度，让甩保安成为一种技巧而非纯拼速度。
- **飞扑有蓄力预警**：危险可预测、可应对（翻滚/变向破解），死亡让玩家觉得"是我没躲好"而非"被偷"。

### 6.4 为什么要"可走动跑道"包住球场
真实球场白线外还有缓冲跑道。把**可走动世界（WORLD）**做得比**画白线的球场（FIELD）**大一圈（每边 110px），消除了"看着是草地却走不过去"的空气墙违和感；同时球员只在白线内踢球，主角/保安可追到跑道上，层次更真实。

### 6.5 手感优先（Game Feel）
惯性、翻滚无敌帧的顺滑、合影时的震屏+爆闪+纸屑+欢呼、跑动时的轻快弹跳与快步频——这些反馈叠加，目标是让"追到、拍到、甩开"的每一下都有**即时、夸张、爽**的回馈。

## 8. 视觉与音效（氛围 Vibe）

- **像素精灵**：用 8×12 像素模板 + 调色板程序化生成各角色（身体 + 站立/跑动两帧腿部），关闭抗锯齿呈现硬边像素质感。
- **跑动动画**：短步频换帧 + 上下小弹跳（hop），轻快灵活。
- **人声鼎沸**：Web Audio 合成的带通滤波白噪声循环模拟人群嘈杂，音量随机起伏；合影 / 暴动时一次"欢呼"音量 swell。
- **闪光灯**：看台随机出现的相机闪光粒子；合影成功时全屏爆闪（球星金闪）。
- **纸屑**：合影成功在目标处爆发彩色纸屑。
- **解说旁白**：开场"有球迷冲场了！他tm疯了吗！"；每次合影随机一句（"观众为你振臂高呼！""比法国超跑还快！""观众里最锋利的剑！"等）；每 10 人里程碑播报。
- **看台助威字幕**：偶发"OLE OLE!""VAMOS!"等横幅文字气泡。
- **危机反馈**：被保安逼近时屏幕边缘泛红（越近越红）。
- **震屏**：合影成功时短暂 Screen Shake。

## 10. 技术架构

- **渲染**：HTML5 Canvas 2D，`requestAnimationFrame` 主循环，固定步长归一化（dt 基于 60fps）。
- **数学**：自实现 `Vec2`（加减、缩放、归一化、距离），用于移动、碰撞、摇杆与所有 AI 向量运算。
- **相机 / 缩放**：`worldToScreen()` 统一做世界→屏幕坐标变换并应用 ZOOM；视野外实体跳过绘制。
- **物理**：所有移动单位采用"速度向目标速度插值"的惯性模型 + 边界夹紧（主角/保安夹到 WORLD，球员夹到 FIELD）。
- **输入**：键盘事件 + Touch/Mouse 虚拟摇杆与按钮，统一汇成移动向量与动作标志。
- **代码组织**：核心逻辑抽取为共享的 `game.js` + `style.css`，被正式版与调试版共同引用——**调试的就是真实游戏代码**。

### 文件结构
```
pitch-invader/
├── index.html      正式版（引用 style.css + game.js）
├── debug.html      调试版（同样引用 game.js，外加调试面板）
├── game.js         全部游戏逻辑（IIFE 封装，尾部挂 window.GAME_DEBUG）
├── style.css       共享样式
└── .claude/launch.json  本地预览静态服务器配置
```
部署：推送到 GitHub，由 GitHub Pages 自动构建发布。

## 11. 调试系统（debug.html）

左上角可折叠面板，与正式版共用 `game.js`，通过 `window.GAME_DEBUG` 控制：

- **实时状态**：FPS、分数、合影数、狂热、保安/球员数量、下次升级阈值、无敌/时速/缩放。
- **快捷按钮**：开始/重开、无敌、慢动作 ×0.3 / 正常 / 快进 ×2、+1000 分、强制升级、生成保安、生成精英、清空保安、生成球星、秒拍最近、触发暴动、显示判定圈、缩放视野。
- **参数滑块（实时生效）**：主角/球员/保安/精英惯性、主角速度、冲刺倍率、保安/精英速度比、球星/普通球员速度、球星/普通体力、镜头缩放。
- **测试钩子**：`step(n)` 手动推进若干帧、`renderOnce()` 强制渲染一帧、`movePlayerTo`、`chaseNearest`、`spawnPlayerNear`、`inspectNearest`——用于在无 rAF 环境下逐项验证逻辑。

## 13. 已知限制与未来可拓展方向

- 当前为纯前端单机，无存档 / 排行榜——可接入 localStorage 最高分或后端榜单。
- 音效为程序化合成，未使用音频素材——可加入解说语音、合影快门声、被捕警报。
- 球员 AI 为"逃跑 + 随机游走"，未实现真实传球 / 战术跑位——可作为风味增强项。
- 升级池仅 4 项——可扩展更多 Roguelite 词条（如磁吸合影、翻滚留残影诱饵、短暂隐身等）。
